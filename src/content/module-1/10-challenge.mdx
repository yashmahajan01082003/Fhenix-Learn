# Challenge: Encrypted Hello World


In this challenge, you will create your first privacy-preserving smart contract that stores encrypted data.

## Learning Objectives

This challenge reinforces:
* Encrypted state storage using `euint32`
* Converting encrypted inputs (`InEuint32`) to compute types
* Understanding that storage contains handles, not plaintext
* Basic FHE operations

## Goal

Create a contract that:
1. Stores an encrypted `euint32` value
2. Allows setting the value from encrypted input
3. Demonstrates that the stored value is encrypted (a handle, not plaintext)

## Steps

1. **Complete the `set` function:**
   - Convert `encryptedInput` from `InEuint32` to `euint32` using `FHE.asEuint32()`
   - Store the result in `val`

2. **Understand what's stored:**
   - The `val` variable stores a `uint256` handle (reference)
   - The actual ciphertext is stored off-chain in the ciphertext registry
   - This is why FHE is practical - handles are small, ciphertexts are large

3. **Test your contract:**
   - Deploy the contract
   - Call `set()` with an encrypted value
   - Call `get()` - notice it returns an `euint32` handle, not a plaintext value

## Key Concepts

* **Encrypted state:** `euint32` represents an encrypted value stored as a handle
* **Input conversion:** `InEuint32` wrappers must be converted using `FHE.asEuint32()`
* **Handle vs ciphertext:** On-chain storage contains handles (uint256), ciphertexts are off-chain
* **Privacy guarantee:** Even if someone reads the storage slot, they only see a handle, not the actual value
